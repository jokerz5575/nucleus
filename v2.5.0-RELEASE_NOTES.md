# 🎉 Clinvio Nucleus v2.5.0: Multi-Database Support Release

This release introduces **enterprise-grade multi-database support** to Clinvio Nucleus Community Edition while maintaining 100% backward compatibility. SQLite remains the default database—no breaking changes for existing users.

## ✨ Major Features

### 🗄️ Multi-Database Support (NEW)
Clinvio Nucleus now supports **5 production-ready databases** out of the box:

- **SQLite** (default) - Perfect for development, embedded, and single-file deployment
- **PostgreSQL 16+** - Enterprise-grade relational database
- **MySQL 8.3+** - Popular web application database
- **Oracle 21+** - Enterprise legacy system support
- **SQL Server 2022+** - Microsoft ecosystem integration

### 🔧 Easy Configuration
Switch databases with a single property:

```yaml
clinvio:
  persistence:
    type: postgresql
    postgresql:
      host: localhost
      port: 5432
      database: clinvio
      username: postgres
      password: secret
```

Or environment variables:
```bash
export CLINVIO_PERSISTENCE_TYPE=postgresql
export CLINVIO_PERSISTENCE_POSTGRESQL_HOST=localhost
```

### 📦 Zero Breaking Changes
- All existing code works without modification
- SQLite is the default (backward compatible)

## 🚀 Getting Started

### For Existing Users
No action required! Your SQLite setup continues to work as before.

### For New Multi-Database Deployments

#### PostgreSQL
```bash
# application.yml
clinvio:
  persistence:
    type: postgresql
    postgresql:
      host: prod-postgres.example.com
      port: 5432
      database: clinvio_prod
      username: clinvio_user
      password: ${DB_PASSWORD}
```

#### Cloud Deployment
```bash
# .env or environment variables
export CLINVIO_PERSISTENCE_TYPE=postgresql
export CLINVIO_PERSISTENCE_POSTGRESQL_HOST=db.c.clinvio.internal
export CLINVIO_PERSISTENCE_POSTGRESQL_PORT=5432
export CLINVIO_PERSISTENCE_POSTGRESQL_DATABASE=clinvio
export CLINVIO_PERSISTENCE_POSTGRESQL_USERNAME=postgres
export CLINVIO_PERSISTENCE_POSTGRESQL_PASSWORD=secret
```
