Clinvio Nucleus

Community edition — Java framework for secure, server-rendered web applications with AES-256-GCM encryption, HTMX, and 23 UI components

v2.3.0 MIT License Java 21+ Spring Boot 3.3 8 Modules 160+ Tests

Introduction

Clinvio Nucleus Community is a free, open-source Java framework for building secure, server-rendered web applications. It combines Thymeleaf templates with HTMX for partial page updates, AES-256-GCM field-level encryption for sensitive data, and a modular architecture across 8 MIT-licensed modules.

AES-256-GCM

Field-level encryption with @SensitiveData

23 UI Components

Buttons, forms, tables, cards, toasts, and more

HTMX

Server-rendered HTML with partial page updates

JWT Security

Authentication, RBAC, CORS, CSRF, token refresh

Workflow Engine

State machine for entity lifecycle management

SQLite

Zero-config production database

This is the Community edition (MIT). Professional and Enterprise editions with additional modules (PostgreSQL, Redis, S3, AI, compliance) are available at clinvio.hu/nucleus.

Quick Start

1. Add Dependency

<dependency> <groupId>hu.clinvio</groupId> <artifactId>clinvio-nucleus-spring-boot-starter</artifactId> <version>2.3.0</version> </dependency>

2. Configure

# application.properties spring.datasource.url=jdbc:sqlite:./data/app.db spring.datasource.driver-class-name=org.sqlite.JDBC spring.jpa.hibernate.ddl-auto=update # Encryption key (use environment variable in production) clinvio.persistence.encryption-key=${CLINVIO_ENCRYPTION_KEY}

3. Create Entity

@Entity public class Patient extends BaseEntity { @SensitiveData @Convert(converter = EncryptedStringConverter.class) private String name; @SensitiveData @Convert(converter = EncryptedStringConverter.class) private String ssn; private String patientNumber; // Not encrypted }

4. Run

@SpringBootApplication public class App { public static void main(String[] args) { SpringApplication.run(App.class, args); } }
That's it. Your entity has AES-256-GCM encrypted fields, auto-generated CRUD endpoints, and HTMX-ready UI components.

Architecture

Clinvio Nucleus consists of 8 modules with a clean dependency hierarchy:

clinvio-nucleus (parent) ├── clinvio-nucleus-core Component model, dialect, renderer ├── clinvio-nucleus-components 23 UI components (button, card, form, table) ├── clinvio-nucleus-themes CSS design system (clinvio-theme.css) ├── clinvio-nucleus-htmx HTMX request/response handling ├── clinvio-nucleus-persistence Entities, AES encryption, SQLite dialect ├── clinvio-nucleus-business Services, controllers, workflow engine ├── clinvio-nucleus-security JWT auth, @Secured, CORS, CSRF └── clinvio-nucleus-spring-boot-starter Auto-configuration

Request Flow

Browser ──► HxRequestFilter │ ▼ DispatcherServlet │ ▼ Controller (CvBaseCrudController) │ ▼ Service (CvBaseService) │ ▼ Repository (CvBaseRepository) │ ▼ EncryptedStringConverter ──► SQLite │ ▼ Thymeleaf + ClinvioDialect │ ▼ HTML Response

Entities & Encryption

BaseEntity

All entities extend BaseEntity, which provides audit fields and soft delete:

@Entity public class Order extends BaseEntity { private String orderNumber; // Inherits: id, createdAt, updatedAt, deletedAt }
FieldTypeDescription
idLongAuto-generated primary key
createdAtLocalDateTimeSet automatically on persist
updatedAtLocalDateTimeSet automatically on update
deletedAtLocalDateTimeSoft delete (null = active)

Field Encryption

Mark String fields with @SensitiveData + @Convert for AES-256-GCM encryption per NIST SP 800-38D:

@SensitiveData @Convert(converter = EncryptedStringConverter.class) @Column(nullable = false) private String customerName; // Encrypted in database private String orderNumber; // NOT encrypted
Both annotations are required. JPA does not process @Convert on meta-annotations. Encrypted fields cannot be searched with SQL LIKE queries.

Encryption Details

PropertyValue
AlgorithmAES-256-GCM (NIST SP 800-38D)
Key DerivationPBKDF2 with 600,000 iterations (OWASP 2023)
IV12-byte random, generated per encryption
GCM Tag128-bit authentication tag
SaltConfigurable via clinvio.persistence.encryption-salt

Repositories

Extend CvBaseRepository for soft-delete-aware data access:

@Repository public interface OrderRepository extends CvBaseRepository<Order, Long> { // CvBaseRepository provides: // findAllActive(), findActiveById(), softDelete(), countActive() @Query("SELECT o FROM Order o WHERE o.deletedAt IS NULL AND o.orderNumber LIKE %:q%") Page<Order> search(@Param("q") String query, Pageable pageable); }

Services

Extend CvSearchableService for full CRUD with search support:

@Service @Transactional(readOnly = true) public class OrderService extends CvSearchableService<Order, Long> { public OrderService(OrderRepository repo) { super(repo, repo); } @Override public Page<Order> search(String query, Pageable pageable) { if (query == null || query.isBlank()) return findAll(pageable); return ((OrderRepository) repository).search(query, pageable); } @Override protected String getEntityName() { return "Order"; } }

Controllers

Extend CvBaseCrudController for auto-generated CRUD views:

@Controller @RequestMapping("/orders") public class OrderController extends CvBaseCrudController<Order> { public OrderController(OrderService service) { super(service); } @Override protected String getListTemplate() { return "orders/list"; } @Override protected String getDetailTemplate() { return "orders/detail"; } @Override protected String getEntityName() { return "Order"; } @Override protected String getEntityPath() { return "/orders"; } @Override protected CvDataTable<Order> buildDataTable(Page<Order> page) { return new CvDataTable<Order>() .addColumn("orderNumber", "Order #", true) .addColumn("customerName", "Customer", true) .data(page.getContent()) .searchable(true).pageable(true).pageSize(page.getSize()); } @GetMapping public String listView(Model model, Pageable pageable) { return super.listView(model, pageable); } @GetMapping("/{id}") public String detailView(@PathVariable Long id, Model model) { return super.detailView(id, model); } }

REST API Controller

@RestController @RequestMapping("/api/orders") public class OrderApiController extends CvBaseApiController<Order> { public OrderApiController(OrderService service) { super(service); } @Override protected String getEntityName() { return "Order"; } @Override protected String getEntityPath() { return "/api/orders"; } } // Auto-generates: // GET /api/orders — List with pagination // GET /api/orders/{id} — Get by ID // POST /api/orders — Create // PUT /api/orders/{id} — Update // DELETE /api/orders/{id} — Soft delete // GET /api/orders/count — Count

Workflow Engine

Define state machines for entity lifecycle management:

WorkflowEngine<OrderStatus> workflow = WorkflowEngine.of(OrderStatus.class) .transition(PENDING, PROCESSING, CANCELLED) .transition(PROCESSING, SHIPPED, CANCELLED) .transition(SHIPPED, DELIVERED) .onTransition(PROCESSING, SHIPPED, (from, to) -> { notificationService.send(userId, "Shipped", "Order shipped"); }) .build(); // Validate transition workflow.validate(currentStatus, newStatus); // throws CvBusinessException if invalid // Execute with callbacks workflow.transition(order, newStatus, order::setStatus);

StatusAware Interface

Entities must implement WorkflowEngine.StatusAware:

@Entity public class Order extends BaseEntity implements WorkflowEngine.StatusAware<OrderStatus> { private OrderStatus status; @Override public OrderStatus getStatus() { return status; } public void setStatus(OrderStatus status) { this.status = status; } }

HTMX Integration

Clinvio provides HxResponse for HTMX-aware responses and HxRequestFilter for request detection.

HxResponse

// Toast notification + redirect return HxResponse.builder() .trigger("cv:toast", Map.of("message", "Created", "type", "success")) .redirect("/orders/" + id) .build(); // Fragment swap (partial page update) return HxResponse.builder() .swap(HxSwap.OUTER_HTML) .fragment("orders/fragments :: status-badge") .model(Map.of("status", newStatus)) .build(); // Push URL without reload return HxResponse.builder() .pushUrl("/orders/" + id) .fragment("orders/fragments :: detail") .model(Map.of("order", order)) .build();

HxSwap Options

ValueDescription
INNER_HTMLReplace inner HTML (default)
OUTER_HTMLReplace entire element
BEFORE_BEGINInsert before element
AFTER_BEGINPrepend inside element
BEFORE_ENDAppend inside element
AFTER_ENDInsert after element

Security (JWT)

Full JWT authentication with Spring Security integration, RBAC, CORS, CSRF, and token lifecycle management.

Authentication Flow

1. Client sends credentials to your login endpoint 2. Your endpoint validates credentials and calls CvJwtService.generateToken() 3. Client stores JWT and sends it in Authorization: Bearer <token> header 4. CvJwtAuthenticationFilter validates token and sets SecurityContext 5. @Secured annotation checks roles on protected endpoints

Generate Token

@Autowired CvJwtService jwtService; // Generate access token String token = jwtService.generateToken(userId, username, "ADMIN", "USER"); // Generate refresh token (7-day expiry) String refreshToken = jwtService.generateRefreshToken(userId, username); // Validate boolean valid = jwtService.validateToken(token); // Extract claims String userId = jwtService.getUserId(token); String username = jwtService.getUsername(token); String[] roles = jwtService.getRoles(token);

Role-Based Access

@Secured(roles = {"ADMIN"}) @GetMapping("/admin/users") public String adminUsers(Model model) { ... } @Secured // Just requires authentication @GetMapping("/profile") public String profile(Model model) { ... }

Auth Endpoints

EndpointMethodDescription
/api/auth/refreshPOSTExchange refresh token for new access token
/api/auth/logoutPOSTBlacklist current token
/api/auth/csrfGETGet CSRF token for HTMX requests

Refresh Token

// Client sends refresh token POST /api/auth/refresh { "refreshToken": "<refresh-token>" } // Response { "accessToken": "<new-access-token>", "tokenType": "Bearer" }

Logout

// Client sends current token POST /api/auth/logout Authorization: Bearer <token> // Response { "message": "Logged out successfully" } // Token is blacklisted and will be rejected on subsequent requests
